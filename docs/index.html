1. Varying Spring Constant

By Varying the spring constant ks, we could observe some interesting effects, for example, for a small ks, such as 1.0, the cloth is very elastic that it behaves like thin silk; for large ks, the cloth becomes very "stiff", that it's more like a piece of paper, this is because the effect is controlled by the "stiffness" of the spring.

2. Varying Density

By changing the density of the point mass, the mass of the cloth changes. As we will see below, the top of the cloth has different behavior: the larger the density is, the more concave the top edge of the cloth is. This is mainly because the higher the density is, the more mass concentrates at the middle, pulling the cloth down to the bottom.

3. Varying Damping Factor

When changing the damping factor, we are essentially changing how much the velocity of the point masses decay. The larger the damping factor is, the more tension there are within the cloth; the smaller the damping factor is, the more flexible the cloth is. Also it oscillates more when the damping factor is small. This is because damping controls the friction experienced by the point masses.

Part 3: Handling collisions with other objects

Collision is a very common physical effect that happens quite often in physical simulation. In order to simulate collision, we perform the following: if the point mass goes past a surface, we "bump" the point mass back to the point where it's tangent to the surface.

Part 4: Handling self-collisions

One simple but crucial error that the current model is that point masses could cross over with each other. We could fix this issue by enforcing another constraint: those point masses that are less than a certain threshold distance apart must be separated, aka all pairs of point masses must be 2 * thickness distance apart.

Part 5: Shaders


Similar to ray tracing, we wish to model the light effect on the objects. We use the OpenGL shader program, which essentially runs the shading algorithm in parallel in GPU. The shader program is a kind of program that could render different effects. Each instance of shader program runs independently and could be parallelized. The .vert file is a vertex shader, this shader program apply transforms to vertices, and we could modify the position of a given vertex in this program. On the other hand, the .frag file is a fragment shader program, and this program takes in geometric attributes, and outputs a color value which is then used by the rasterization algorithm.
